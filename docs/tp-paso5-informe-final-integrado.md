## 2. Introduccion

En este trabajo practico se analiza el sistema IFTS15 desde la perspectiva del aseguramiento de calidad del software. Para ello, el grupo se organizo como un area de calidad y aplico conceptos, normas y marcos metodologicos vistos en la materia con el fin de evaluar tanto el producto desarrollado como el proceso que hizo posible su construccion.

El objetivo del informe es determinar en que medida el sistema cumple criterios de calidad, identificar riesgos y debilidades, y proponer mejoras concretas para una futura evolucion del proyecto.

## 6. Conclusion grupal

Texto base sugerido:

Como conclusion grupal, se considera que IFTS15 es un sistema funcional, con alcance real y utilidad concreta dentro de su contexto academico. El analisis realizado muestra que el producto presenta fortalezas importantes en adecuacion funcional y cobertura de necesidades institucionales, pero tambien evidencia debilidades vinculadas al proceso de desarrollo, la formalizacion de requisitos, la estrategia de testing y la madurez general de aseguramiento de calidad.

Si el proyecto se volviera a desarrollar hoy, el grupo propone trabajar con mayor definicion de requisitos, mejor documentacion, roles mas claros, control de cambios, testing continuo y una metodologia agil con practicas de calidad integradas.

## 7. Reflexion sobre el trabajo en equipo

Texto guia sugerido:

La conformacion del area de calidad permitio observar el sistema desde una perspectiva distinta a la del desarrollo original. El trabajo por roles ayudo a distribuir responsabilidades y a enfocar el analisis desde distintos criterios complementarios. Al mismo tiempo, puso en evidencia que la calidad del software no depende solo del codigo, sino tambien de la organizacion del trabajo, la documentacion, la validacion y la mejora continua.
